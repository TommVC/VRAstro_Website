Selection by Deformation

This selection method makes use of a sphere that the user can deform using the VR controllers. Deformation can be thought of as moulding the sphere like a ball of clay into the shape the user desires. In many cases, the sub-regions of interest in these data cubes are strangely shaped. Hence, having a large degree of control in creating the shape by deforming can be useful for selecting out these regions.

The images to the right display a user selecting out a region with their deformed sphere.

Deformation Model

Free-Form Deformation

The process hidden behind a user tugging on the sphere and moulding it is quite complex, with fine details explored in the final paper. Generally, the process involves embedding the sphere inside a lattice of control points. [ A ] in the diagram to the right shows the sphere being embedded in three different lattices. The only thing different about these lattices are the number of control points in each direction.

When a user pulls the surface of the sphere in a given direction, the location and direction of movement of the users hand are sent to the deformation model. This causes a shift in the control points, which then allows the sphere to be moulded relative to the shift in the control points. The number of control points in the lattice affects how much of the sphere gets deformed, this is shown in [ B ].

Since the user needs to make more than one deformation, the lattice needs to adapt to the new shape to ensure it fits inside. [ C ] displays the lattice dynamically changing by incorporating more (green) points to the lattice.

Above are some box and whisker diagrams representing the data collected from the experiment we conducted. The only metric missing is workload, as this is divided into a few sub-metrics and for the sake of space has been omitted. These results are present in the final paper. The diagrams above show that there is really not much difference between the desktop method and the deformation method. The only metric with a significant difference is efficiency; we can see that tasks were performed slightly quicker overall with the desktop method. For all other metrics there is no significance between between the two methods. Hence, at this point, we cannot say whether a virtual reality environment provides a better environment for selection than a desktop. It should be noted that the deformation method did have some bugs which got in the way of experiments. I am confident that if these were resolved, along with some extra functionality added to the model, as well as better experiment conditions, that better results would be produced.
