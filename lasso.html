Lasso Selection.

Lasso Overview.

Usage of the Lasso method:
The user first positions the "whiteboard" for them to draw on. Once locked the user will notice a second whiteboard behind the first. The back whiteboard can be moved towards or away from the user, therefore controlling the depth.
With the boards properly positioned, a user can begin to draw with the controller by pointing at the board and holding a button down. Users can reset input at any point with the menu reset button. Once happy with the input shape, users can generate the lasso shape ready for selection. By default the method adds to a selection, however a subtractive mode can be toggled between. This allows users to add and scalpt away to their desired selection.

Components of lasso method:
Planes: Used to position the system and adjust the length of the lasso.
Shape Creation: This used line renderers to construct the perimeter of the shape on both planes. Mesh Creation: Created a mesh later used to identify whether data points are inside or outside of the lasso. Inside-Outside Test:

Background and context:

The premise of lasso selection typically lends itself to 2D environments. This tendency is due to the simplicity of how regions of interest are defined. This can be observed in Adobe Photoshop when using their lasso tool to select a desireable region. The user input in 2D lasso selections is a 1:1 mapping to the perimeter of the shape making up a selection. However, with 3D lasso selections there lacks reliable 3D user input that is also easy to use. A common work around is to record a 2D input (imagine drawing on a whiteboard) and then projecting the drawn shape into the direction the user is facing. Another alternative are hybrid systems that make use of drawing tablets and virtual reality headsets to record the input. For lasso volumetric selection to be of added value, the length of the projection needs to controlled by the user. Otherwise lasso selections would result in excessive or insufficient selections. Furthermore, the ability to iteratively define selections with varying inputs would result in greater user control.

Objectives and implemented features:

The aim of this project was to determine weather volumetric selection through a lasso method in virtual reality provided an improvement (increased accuracy, efficiency, usability and decreased workload) in data selection in comparison to a desktop environment.

Implemented features:
• Adjustable depth of lasso into the screen or third dimension.
• The lasso system records and resets user input.
• It also edits the user input to close the shape and remove overlappings.
• Iterative use of shapes to add to a selection.
• Can iteratively add and subtract to and from existing selections.

Results:

To carry out the objective of this project, a study was conducted with 25 users completing four selections . Each user was given training with the iDaVIE-V system in VR. The users' time in VR was limited to an hour to avoid fatigue, this also constrained our training to five minutes per method, and ten minutes per selection to also account for the inital training. Method performance was evaluated based on accuracy, efficiency, usability and workload. Accuracy was measured by the F1 score and MCC, efficiency by completion time, usability with System Usability Scale questionnaires and workload was measured with the NASA Task Load Index. After conducting a statistical analysis, it was proven that the lasso method performed significantly worse than the desktop benchmark across all metrics. This was in part due to incomplete code creating bugs that frustrated users and caused incorrect selections.
